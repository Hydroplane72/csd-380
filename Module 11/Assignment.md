# Proving Compliance in Regulated Environments
## Description
Amazon Web Services is creating processes that allow regulators the abilty to audit systems without requiring direct developer access to the systems. This is done by providing telemetry data from production systems to regulators. The telemetry data is collected from production systems and sent to a secure location where regulators can access it. 

## Main points
  - All major actions are logged
  - All auditable logs are viewable for regulators
  - The main goal is to prevent, detect, and correct problems

# Relying on Production Telemetry for ATM Systems
## Description
The article discusses how production systems should not just rely on code reviews to detect fraud. Instead, production monitoring tools should be used in addition to automated testing, code reviews, and approvals.

## Main points
  - Code reviews alone are not enough
  - All code review tools should be used in conjunction with production monitoring tools
  - Fraud can still happen even if code reviews are done correctly, so production monitoring tools should be used to detect fraud in production systems
